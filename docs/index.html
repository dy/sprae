Sprae Framework

Intro

Sprae is a minimal reactive framework for modern web apps. Inspired by Alpine.js, it delivers declarative power in ~2KB, no build step required.

Example

<script src="https://unpkg.com/sprae"></script>
<div :init="count=0">
  <button @click="count++">+</button>
  <button @click="count--">-</button>
  Count: <span :text="count"></span>
</div>

Principles

  • Minimal API, tiny footprint
  • Progressive enhancement by default
  • Direct HTML binding — no compilation

Reference

Directives: :init, :text, @event, :class, :style, :html, :if, :for

Examples

  • :if — toggle element visibility
  • :for — list rendering
  • :class — conditional classes
  • :style — dynamic inline styles
  • :html — render raw HTML

FAQ

Why another framework? For when you need small, declarative reactivity without a complex build system.

Why Sprae? Smaller than Alpine, simpler than Vue, pure HTML binding.

Why not Alpine.js, Petite-Vue, Lucia? Sprae is lighter, with zero extras.

Why not vanilla JS? Avoids repetitive boilerplate for reactive UIs.

Who cares about progressive enhancement? Anyone valuing speed and resilience.

Does it scale? Yes, within its scope for UI state.

Is it just a toy? No — production ready for small/medium apps.

Complex state? Possible with modular architecture.

No build outdated? No, it’s ideal for many projects.

new Function unsafe? Safe with trusted templates.

Next/React? Works, but not the primary target.

Components? HTML is your component model.
