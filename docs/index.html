spræ

Hydration for DOM tree

Easy progressive enhancement for HTML / JSX.
Add UI interactions with inline reactive attributes.
<script src="https://unpkg.com/sprae"></script>
<div :init="count=0">
  <button @click="count++">+</button>
  <button @click="count--">-</button>
  Count: <span :text="count"></span>
</div>

Principles

  • Minimal API, tiny footprint
  • Progressive enhancement by default
  • Direct HTML binding — no compilation

Reference

Directives: :text, :on, :class, :style, :scope, :if, :each

FAQ

Why another framework? No need for a permit to reinvent the wheel.

Why Sprae? It hydrates your HTML tree like Alpine dew.

Why not Alpine.js, Petite-Vue? Fewer footprints, lighter gear.

Why not vanilla JS? Because `querySelector` is wrong mantra.

Progressive enhancement: who cares? Anyone on 3G.

Does it scale? As far as your CPU does.

Just a toy? If production counts as a toy.

Complex state possible? If you can handle it, but don't overthink.

Signals overhyped? They are just fancy variables.

No build step outdated? Your 12-min build pipeline is outdated.

new Function unsafe? Only if your HTML comes from strangers.

Why not Next/React? Because not every button needs a megabyte of runtime.

Components support? Already there — called HTML tags.

Why not vibecode? You can vibe-sprae instead. See what we did here?

Ecosystem? Yes. The web, since 91.

TypeScript? Soon™.

Backing by a company? No. Just regular devs. You can join!
